# MVideo Inventory Management System

Программа на Kotlin для обработки операций с товаром (поступления и продажи) и расчета остатков на складе.

## Описание

Приложение принимает CSV-файл с операциями поступления и продажи товаров, обрабатывает их в порядке очередности и генерирует выходной CSV-файл с итоговыми остатками.

### Алгоритм продажи

При продаже товар списывается в порядке лексикографической сортировки по ID товара (товар с более "высоким" рангом продается первым). Если товара недостаточно, разница записывается с отрицательным значением на товар с наивысшим рангом в группе.

## Требования

- Java 21+ (или более поздняя версия)
- Gradle 8.0+

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/IntFxZen/MVideo.git
cd MVideo
```

### 2. Сборка проекта

```bash
./gradlew build
```

### 3. Запуск приложения

```bash
./gradlew run --args="input.csv output.csv"
```

Или в Windows:
```cmd
gradlew.bat run --args="input.csv output.csv"
```

**Параметры:**
- `input.csv` - путь к входному CSV-файлу с операциями
- `output.csv` - путь к выходному CSV-файлу с результатами

## Формат входного файла

Рекомендуемый: разделитель `;`, без заголовка. Строки:
- Поступление: `groupId;productId;quantity` (3 поля)
- Продажа: `groupId;quantity` (2 поля)

Пример:
```
electronics;phone_A;50
electronics;40
```

Выход
CSV с заголовком: `group_id,product_id,quantity` (запятая)

Допущения
- Количества — целые числа
- Ранг — лексикографический порядок по productId
- При продаже из пустой группы создаётся `unknown_product` с отрицательным остатком
- Парсер поддерживает fallback на `type,group_id,product_id,quantity`

## Формат выходного CSV

Выходной файл содержит итоговые остатки по всем товарам:

```
group_id,product_id,quantity
group_A,product_1,5
group_A,product_2,10
group_B,product_5,-5
group_C,unknown_product,-8
```

### Структура:
- **group_id**: идентификатор группы
- **product_id**: идентификатор товара
- **quantity**: итоговый остаток (может быть отрицательным)


## Логирование

Приложение использует SLF4J с Logback для логирования:

- **INFO** - основные события (начало/конец обработки, кол-во операций)
- **WARN** - предупреждения (отсутствие товаров, недостаточный остаток)
- **DEBUG** - детальная информация о каждой операции

Уровень логирования можно изменить в файле `src/main/resources/logback.xml`.

## Архитектура

Проект следует принципам чистой архитектуры и разделен на слои:

```
src/main/kotlin/com/nn/mvideo/
├── Main.kt                    # Точка входа приложения
├── model/                     # Слой моделей
│   ├── Operation.kt           # Sealed class для операций
│   └── ProductBalance.kt      # Data class для остатков
├── service/                   # Бизнес-логика
│   └── InventoryService.kt    # Сервис управления инвентарем
└── io/                        # Слой ввода-вывода
    ├── CsvReader.kt           # Чтение CSV
    └── CsvWriter.kt           # Запись CSV
```

## Допущения

1. **Формат CSV**: Используется разделитель `,` (запятая), header обязателен
2. **Тип данных**: Все количества - целые числа
3. **Ранг товаров**: Определяется лексикографической сортировкой (alfabetical order)
4. **Отрицательный остаток**: Разница при недостаточном товаре записывается на товар с наивысшим рангом
5. **Несуществующая группа при продаже**: Создается запись с товаром `unknown_product` и отрицательным остатком
